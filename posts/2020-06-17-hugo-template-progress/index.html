Progress on Figuring out Hugo's template system

Jun 17, 2020
2 min read

Today, finally I managed to get the page showing the way I originally envisioned.

I wanted to have the links on the left sidebar to be the content from the /content/pages/ folder, while the list on the right (main) section on the home page to show only the pages from the /content/posts/ folder. Just like a regular blog.

The trick lied in modifying two template files: layouts/_default/baseof.html and layouts/_default/list.html

The first is the overall template for the whole site. This is where the left sidebar lives on my template. Here’s the modification I made:

<div>
  {{ range (where .Site.RegularPages "Section" "pages") }}
    <div><a href={{ .RelPermalink }}>{{ .Title }}</a></div>
  {{ end }}
</div>

This lists all the regular (single) pages of the site which reside in the “pages” Section

And the second file (layouts/_default/list.html) is the template for the list pages. Here I needed to put in a conditional to handle things differently on the home page.

{{ $pages := .Paginator.Pages}}
{{ if .IsHome }}
  {{ $pages = where .Site.RegularPages "Section" "posts"}}
{{ end }}
{{ range $index, $element :=  $pages}}
  ...
  ...
{{ end }}

I will probably need to look into the pagination of the post pages. The original one used a .Paginator.Pages reference, which I didn’t find much information on so far. But so far there’s only a few posts, so I’ll tackle the paginations when it presents a problem (after probably a hundred posts - haha :D )

BTW, I found out that the metadata in the beginning of the markdown files is called “front matter”. With that keyword I was able to find much more information :)

So, slowly but surely, I’m mastering this. It’s quite interesting to see how far I’ll get with it, if I keep figuring out one thing every day.
