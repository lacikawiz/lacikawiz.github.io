My Fist Post with Hugo

Jun 14, 2020
2 min read

The first post with Hugo

This Sunday morning I woke up early and I decided to try if I can put together a static website with Hugo and publish it with Github pages. I had been thinking about this for about a year now, since I read some articles about it. I’m very interested in making very fast loading sites, which are also simple under the hood. I also wanted to make a site where I can publish my thoughts without the noise of social networking sites. It’s like a personal space that I fully own and make it in any way I want.

I had a Github pages site setup already so that made the process a little bit shorter. Pretty much everything went smoothly until I hit some roadblocks with the template.

I chose a template called m10c. I tried to change the colors in it by following the instructions on their page. And, they just didn’t want to change!

After some digging I found out that this template uses SCSS for the styling, and the Hugo executable I downloaded was not the “extended” version. Which means it couldn’t handle SCSS. After downloading the extended version of Hugo. The changes finally started taking effect.

The next bump was also related to the template. It was using full URLs, and because I had the Github page URL in my config.toml file it added the Github page URLs even when I was running the site locally. After some Googling I found what I need to change. I replaced all .Permalink references to .RelPermalink and also replaced the home page references to just /. Now, everything is working nicely.

I love the fact that generating the site takes only milliseconds! And loads super fast. I’m sure it gets somewhat slower when I start adding.

Now, I’m figuring out how to add content. I found that it’s not easy to find the documentation on what needs to go into the header section of a markdown file. Maybe I don’t know the lingo? What is it actually called?

I admit I didn’t sit down reading through the Hugo documentation. I just jumped into it head first. If all else fails I read the documentation :) Making mistakes is a great learning experience.

This is it for now. I’m enjoying this experiment a lot. I could sit and play with this for hours, but the family needs some attention too. Will continue tomorrow.
